# **Types of Cloud Computing (Deployment Models)**

Cloud computing is classified based on **who manages the infrastructure** and **who has access** to the resources. These are called **deployment models**. There are **four main types**: **Public Cloud, Private Cloud, Hybrid Cloud, and Community Cloud**.

---

## **1. Public Cloud**

**Definition:**
A **public cloud** is a cloud infrastructure that is **open to the general public** or a large industry group. Resources such as servers, storage, and applications are owned and managed by a **third-party cloud provider**.

**Key Features:**

* Shared infrastructure among multiple users (multi-tenancy)
* Pay-as-you-go pricing model
* No need to manage hardware or software
* Scalable and elastic

**Advantages:**

* Cost-effective (no upfront investment)
* Easy access from anywhere
* High scalability for varying workloads

**Disadvantages:**

* Limited control over security
* Possible downtime if the provider has issues

**Examples:**

* Amazon Web Services (AWS)
* Microsoft Azure
* Google Cloud Platform (GCP)

---

## **2. Private Cloud**

**Definition:**
A **private cloud** is a cloud infrastructure **dedicated to a single organization**. It can be managed **internally or by a third-party provider**, but the resources are **not shared with other organizations**.

**Key Features:**

* Exclusive access to organization data and applications
* Higher security and control
* Can be on-premises or hosted by a provider

**Advantages:**

* Enhanced security and compliance
* Greater control over resources
* Customizable infrastructure

**Disadvantages:**

* Higher cost (requires maintenance and IT staff)
* Limited scalability compared to public clouds

**Examples:**

* VMware vSphere
* OpenStack Private Cloud
* IBM Private Cloud

---

## **3. Hybrid Cloud**

**Definition:**
A **hybrid cloud** combines **both public and private clouds**, allowing data and applications to be **shared between them**.

**Key Features:**

* Combines the **flexibility of public cloud** and the **security of private cloud**
* Enables data or workload portability between clouds
* Can optimize cost and performance

**Advantages:**

* Flexible and scalable
* Sensitive data can remain on private cloud
* Cost-efficient (use public cloud for peak demand)

**Disadvantages:**

* Complex management
* Possible compatibility issues between public and private clouds

**Examples:**

* Company using **AWS for customer-facing apps** but **private cloud for sensitive data**
* Microsoft Azure with on-premises datacenter integration

---

## **4. Community Cloud**

**Definition:**
A **community cloud** is **shared infrastructure** for a **specific group of organizations** with common goals, security requirements, or compliance needs.

**Key Features:**

* Shared among organizations with **similar requirements**
* Cost is shared across the community
* Managed internally or by a third-party provider

**Advantages:**

* Cost-effective for organizations with similar needs
* Better collaboration among the community
* Compliant with regulatory standards

**Disadvantages:**

* Limited scalability compared to public clouds
* Not suitable for unrelated businesses

**Examples:**

* Government cloud for multiple departments
* Healthcare cloud for hospitals sharing patient data securely
